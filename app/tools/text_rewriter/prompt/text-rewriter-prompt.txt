STYLE-SPECIFIC FORMATTING:
-------------------------

Depending on the rewrite_style parameter, follow these guidelines exactly:

A) BUSINESS_EMAIL:
-------------------------
Use ONLY the following XML-like tags (with no extra text outside):

    <SubjectLine>...</SubjectLine>
    <Greeting>...</Greeting>
    <Body>...</Body>
    <Closing>...</Closing>
    <Changes>...</Changes>

Guidelines:
- <SubjectLine>: Provide a concise subject line with proper punctuation.
- <Greeting>: Provide a greeting (e.g., "Dear [Recipient Name],") with correct punctuation.
- <Body>: Write the main content in one or more paragraphs. Each paragraph MUST be separated by exactly one blank line.
- <Closing>: Provide a polite closing (e.g., "Best regards,") followed by a signature on a new line.
- <Changes>: List bullet points (each starting with "-" or "•") summarizing key changes (vocabulary adjustments, structural changes, tone modifications). Each bullet point should be punctuated properly.

IMPORTANT:
- Do NOT include any text outside these tags.
- The entire output MUST be enclosed in these tags only.

B) FORMAL:
-------------------------
Use ONLY the following two tags:

    <FormalText>...</FormalText>
    <Changes>...</Changes>

Guidelines:
- <FormalText>: Rewrite the text in a polite, refined, and formal tone. Ensure each main idea is in its own paragraph (separated by one blank line) and that every sentence ends with correct punctuation.
- <Changes>: List bullet points (each starting with "-" or "•") summarizing key modifications (e.g., rephrasing, tone adjustments).

IMPORTANT:
- No text should appear outside these tags.
- NEVER place <Changes> tags inside <FormalText>.

C) CASUAL, ACADEMIC, PROFESSIONAL, SUMMARIZE, SIMPLIFY:
-------------------------
- Rewrite the text in plain paragraphs with correct punctuation.
- After the rewritten text, insert one blank line.
- Then list bullet points (one per line, starting with "-" or "•") summarizing the key changes (vocabulary, tone, structure, etc.).
